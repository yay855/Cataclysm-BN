5/**
6 * debugmsg(msg, ...)
7 * varg-style functions: the first argument is the format (see printf),
8 * the other optional arguments must match that format.
9 * The message is formatted and printed on screen to player, they must
10 * acknowledge the message by pressing SPACE. This can be quite annoying, so
11 * avoid this function if possible.
12 * The message is also logged with DebugLog, a separate call to DebugLog
13 * is not needed.
14 *
15 * DebugLog(lev, cl) and DebugLogFL(lev, cl)
16 * The main debugging system. It uses debug levels (how critical/important
17 * the message is) and debug classes (which part of the code it comes from,
18 * e.g. mapgen, SDL, npcs). This allows disabling debug classes and levels
19 * (e.g. only write SDL messages, but no npc or map related ones).
20 * It returns a reference to an output stream. Simply write your message into
21 * that stream (using the << operators).
22 * DebugLog always returns a stream that starts on a new line. Don't add a
23 * newline at the end of your debug message, as it is automatically appended
24 * once the reference goes out of scope.
25 * If the specific debug level or class have been disabled, the message is
26 * actually discarded, otherwise it is written to a log file.
27 * Errors are logged regardless of whether debug class is enabled.
28 * DebugLogFL version also logs source file name and line number.
29 * If a single source file contains mostly messages for the same debug class
30 * (e.g. mapgen.cpp), create and use the macro dbg.
31 *
32 * dbg(lev)
33 * Usually a single source contains only debug messages for a single debug class
34 * (e.g. mapgen.cpp contains only messages for DC::MapGen, npcmove.cpp only DC::NPC).
35 * Those files contain a macro at top:
36@code
37#define dbg(x) DebugLogFL((x), DC::NPC)
38@endcode
39 * It allows to call the debug system and just supply the debug level, the debug
40 * class is automatically inserted as it is the same for the whole file. Also this
41 * adds the file name and the line of the statement to the debug message.
42 * This can be replicated in any source file, just copy the above macro and change
43 * DC::NPC to the debug class to use. Don't add this macro to a header file
44 * as the debug class is source file specific.
45 * As dbg calls DebugLog, it returns the stream reference, its usage is the same.
46 */
