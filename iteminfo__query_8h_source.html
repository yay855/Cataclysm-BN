226 /* The implemented constructors are a bit arbitrary right now. Currently there are
227
228 ( const std::string &bits )

231
232 used to allow simple 'all'/'none' presets or potentially _very_ specific
233 combinations *but* you have to include _every_ bit here so, it'll mostly
234 just be all/none
235
236 ( const iteminfo_query_base &values )
237
238 ( A & (~ B | C) ^ D )
239
240 allows usage of the underlying std::bitset's many bit operators to combine
241 any sort of fields needed
242
243 ( const std::vector<iteminfo_parts> &setBits )
244
245 ( std::vector { iteminfo_parts::Foo, iteminfo_parts::Bar, ... } )
246
247 allows defining a subset with only specific bits turned on
248
249 These should be sufficient to allow _any_ preset to be defined.
250
251 Since those typically _always_ should all be 'static const' performance should
252 not be any issue.
253 */
