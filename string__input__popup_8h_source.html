21/**
22 * Shows a window querying the user for input.
23 *
24 * Returns the input that was entered. If the user cancels the input (e.g. by pressing Escape),
25 * an empty string is returned. An empty string may also be returned when the user does not enter
26 * any text and confirms the input (by pressing ENTER).
27 *
28 * Examples:
29 * \code
30 input = string_input_popup().title("Enter something").query();
31 // shows the input field in window w at coordinates (10,20) up to (30,20).
32 input = string_input_popup().window(w, 10, 20, 30).query();
33 * \endcode
34 *
35 * @param title The displayed title, describing what to enter. @ref color_tags can be used.
36 * @param width Width of the input area where the user input appears.
37 * @param input The initially display input. The user can change this.
38 * @param desc An optional text (e.h. help or formatting information) which is displayed
39 * above the input. Color tags can be used.
40 * @param identifier If not empty, this is used to store and retrieve previously entered
41 * text. All calls with the same `identifier` share this history, the history is also stored
42 * when saving the game (see @ref uistate).
43 * @param max_length The maximal length of the text the user can input. More input is simply
44 * ignored and the returned string is never longer than this.
45 * @param only_digits Whether to only allow digits in the string.
46 */
