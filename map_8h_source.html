362/**
363 * Manage and cache data about a part of the map.
364 *
365 * Despite the name, this class isn't actually responsible for managing the map as a whole. For that function,
366 * see \ref mapbuffer. Instead, this class loads a part of the mapbuffer into a cache, and adds certain temporary
367 * information such as lighting calculations to it.
368 *
369 * To understand the following descriptions better, you should also read \ref map_management
370 *
371 * The map coordinates always start at (0, 0) for the top-left and end at (map_width-1, map_height-1) for the bottom-right.

373 * The actual map data is stored in `submap` instances. These instances are managed by `mapbuffer`.
