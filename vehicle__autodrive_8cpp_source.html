37/*
38 * This file contains code that allows a vehicle to be driven by an in-game character (most
39 * likely the player character) to some remote destination, following a path of adjacent overmap
40 * tile (OMT) coordinates, while avoiding any obstacles along the way.
41 *
42 * The main entry point is do_autodrive(), which is intended to be called from the do_turn() of
43 * a long-running activity. This will cause the driver character to perform one or more driving
44 * actions (as long as they have enough moves), which could be steering and/or changing the
45 * cruise control speed. The driver may also do nothing if the vehicle is going in the right
46 * direction.
47 *
48 * Most of the logic is inside the private implementation class autodrive_controller, which is
49 * instantiated at the beginning of an autodrive activity and cleaned up at the end. This class
50 * caches information about the vehicle, driver and the local terrain in order to minimize the
51 * amount of computation required on each invocation of do_autodrive().
52 *
53 * Upon reaching a new overmap tile, as well as once initially, the autodrive algorithm will
54 * try to find a path that takes the vehicle from its current position through the current
55 * and next OMT in order to reach the 2nd next OMT in the overall path to destination (which
56 * is obtained from driver.omt_path). Thus we always plan ahead 2 OMTs worth of navigation,
57 * but we only follow the first half of the route, then recompute a new route for the next
58 * 2 OMTs. This ensures that we don't drive up to the edge of an OMT only to get stuck due
59 * to a large obstacle that can't be driven around from that position (a dead end) and instead
60 * we always choose to cross OMT boundaries in a position from which we are guaranteed to be
61 * able to continue onward.
62 *
63 * The pathfinding algorithm being employed is A*. The graph being searched is made up of nodes
64 * denoted by x and y coordinates as well as a direction of travel. Since steering changes the
65 * vehicle's direction in increments of 15 degrees there are 360/15 possible orientations. By
66 * limiting navigation to 2 OMTs at a time we have a maximum search space of 2*24*24*24 nodes.
67 * In practice, A* finds a good path after visiting fewer than 50 nodes due to the heuristic
68 * function that estimates which nodes are most likely to lead to the destination (which is
69 * based on proximity to the "exit zone" leading to the next OMT and whether the vehicle is
70 * pointing towards the middle of that zone). Although there can be many paths that we could
71 * take to get through to the next OMT, we just take the first one we find since it tends to
72 * be the most direct and is usually near-optimal. When the way is blocked we'll either give
73 * up after exhausting the search space or after we hit a limit on the search length.
74 *
75 * In order to build the search graph for A*, we consider every possible map tile and orientation
76 * in the search zone and we figure out whether the vehicle's pivot point could be placed there
77 * without the vehicle overlapping some obstacle (a collision-free placement). The pivot point
78 * is the part of the vehicle that moves the least while steering and is usually between the
79 * wheels of the vehicle, closer to the non-steerable ones. The pathfinding problem reduces to
80 * finding a path for the pivot point to follow through the set of possible positions and
81 * orientations (nodes). In order to figure out the possible connections between nodes (the
82 * edges of the graph) we consider a limited set of actions that the driver can perform at
83 * each location: nothing, steer left or steer right (possibly more than once). Given a current
84 * node, current speed and amount of steering we can predict where the vehicle's pivot point
85 * will end up in 1 turn and connect the 2 nodes.
86 *
87 * In order to keep the search space small we only consider paths traveled at constant speed
88 * (otherwise we'd have to add speed as an extra attribute in every node). When we need to
89 * compute a new path we first try to use the maximum safe autodrive speed (which is typically
90 * much less than the vehicle's max speed). If we can't find a path we cut the speed in half and
91 * try again, repeating until we succeed or reach the minimum speed. When the target speed is
92 * different from the current speed of the vehicle we try to estimate how long it will take to
93 * reach that speed and take that into account when simulating the first few edges (they'll be
94 * shorter or longer than the rest).
95 *
96 * Once we have generated a sequence of nodes to follow, we do not invoke A* again until we reach
97 * the next OMT. We simply cache the path and on every subsequent call to do_autodrive() we
98 * check where we are on that path, what node we're supposed to reach next and what action we
99 * should use to get there. However, sometimes the simplified simulation used to compute the
100 * search graph fails to accurately predict where the vehicle will end up. This could be because
101 * there was too much error in an approximation (such as time to accelerate) or the driving
102 * conditions changed (ran over a mound of dirt that interfered with steering). When such a
103 * discrepancy is detected, autodrive will fall back to "safe mode", cutting the speed to the
104 * minimum (1 tile per second) and recomputing the path.
105 *
106 * Since the navigation graph is cached and not updated in response to dynamic obstacles (such
107 * as animals) autodrive will also perform collision detection at every turn, before taking
108 * any action that may end the turn. If a possible collision is detected autodrive will enter
109 * "safe mode", reducing speed and recomputing. If the speed is already at minimum it will
110 * abort instead with a warning message.
111 *
112 * In most cases when autodrive aborts it will not automatically stop the vehicle. Stopping
113 * may be undesirable since nearby enemies could get into the vehicle. It is up to the player
114 * to either stop or continue driving manually for a while. However, when aborting due to a
115 * collision (which happened despite collision detection, i.e. there was a simulation discrepancy
116 * near an obstacle) the brakes will be engaged.
117 *
118 * In order to prevent exploiting the system to navigate in the dark, autodrive will also
119 * perform visibility checks. Not being able to see anything in front of the vehicle will
120 * immediately cancel (or fail to start) autodrive. If the driver has only partial visibility
121 * of the front of the vehicle, safe mode will engage limiting the speed. Furthermore, the
122 * driver needs to either see or have map memory for the map tiles in the current OMT; all
123 * unseen tiles will be treated as obstacles during pathfinding. In practice, this means
124 * that driving at night should only be possible through familiar terrain with limited
125 * lights / night vision or through unfamiliar terrain but with good headlights / excellent
126 * night vision.
127 */

478/*
479 * Computes atan2(dy, dx) in "orientation" units. Optimized to use only integer
480 * arithmetic and table lookups since it's used in the inner loop of A*.
481 */
482// NOLINTNEXTLINE(cata-xy)
483static orientation approx_orientation( int dx, int dy )
484{
485 orientation ret = orientation::d0;
486 if( dy < 0 ) {
487 dx *= -1;
488 dy *= -1;
489 ret += quad_rotation::d180;
490 }
491 if( dx < 0 ) {
492 std::swap( dx, dy );
493 dy *= -1;
494 ret += quad_rotation::d90;
495 }
496 if( dy > dx ) {
497 ret = ret + quad_rotation::d90 - approx_orientation( dy, dx );
498 } else if( dy > 0 ) {
499 static const auto atan_table = [] {
500 constexpr int table_size = 101;
501 std::array<orientation, table_size> table;
502 for( int i = 0; i < table_size; i++ )
503 {
504 table[i] = to_orientation( units::from_radians( std::atan( 1.0 * i /
505 ( table_size - 1 ) ) ) );
506 }
507 return table;
508 }();
509 ret = ret + atan_table[ dy * 100 / dx ];
510 }
511 return ret;
512}

576vehicle_profile vehicle::autodrive_controller::compute_profile( orientation facing ) const
577{
578 vehicle_profile ret;
579
580 auto angle = to_angle( facing );
581 tileray tdir( angle );
582 ret.tdir = tdir;
583 std::map<int, std::pair<int, int>> extent_map;
584 const point pivot = driven_veh.pivot_point();
585 for( const vehicle_part &part : driven_veh.parts ) {
586 if( part.removed ) {
587 continue;
588 }
589 tripoint pos;
590 driven_veh.coord_translate( angle, pivot, part.mount, pos );
591 if( extent_map.find( pos.y ) == extent_map.end() ) {
592 extent_map[pos.y] = { pos.x, pos.x };
593 } else {
594 auto &extent = extent_map[pos.y];
595 extent.first = std::min( extent.first, pos.x );
596 extent.second = std::max( extent.second, pos.x );
597 }
598 }
599 for( const auto &extent : extent_map ) {
600 const int y = extent.first;
601 const int x_min = extent.second.first;
602 const int x_max = extent.second.second;
603 for( int x = x_min; x <= x_max; x++ ) {
604 ret.occupied_zone.emplace_back( x, y );
605 }
606 }
607 for( int part_num : driven_veh.rotors ) {
608 const vehicle_part &part = driven_veh.cpart( part_num );
609 const int diameter = part.info().rotor_diameter();
610 const int radius = ( diameter + 1 ) / 2;
611 if( radius > 0 ) {
612 tripoint pos;
613 driven_veh.coord_translate( angle, pivot, part.mount, pos );
614 for( tripoint pt : points_in_radius( pos, radius ) ) {
615 ret.occupied_zone.emplace_back( pt.xy() );
616 }
617 }
618 }
619 // figure out the maximum amount of displacement that can happen from advancing the
620 // tileray a single step by advancing it a bunch of times and reducing the x and y
621 // components of the displacement to at most 1
622 constexpr int far_advance = 100;
623 tdir.advance( far_advance );
624 const point increment( signum( tdir.dx() ), signum( tdir.dy() ) );
625 // compute the subset of vehicle points that would move to previously unoccupied spots
626 // (a.k.a. "moving first") when the vehicle moves one step; these are the only points
627 // we need to check for collision when the vehicle is moving in this direction
628 const std::unordered_set<point> occupied_set( ret.occupied_zone.begin(), ret.occupied_zone.end() );
629 for( const point &pt : ret.occupied_zone ) {
630 if( occupied_set.find( pt + increment ) == occupied_set.end() ) {
631 ret.collision_points.emplace_back( pt );
632 }
633 }
634 return ret;
635}

645 // check if another vehicle is there; tiles occupied by the current

647 const optional_vpart_position ovp = here.veh_at( pt );

649 // Known corner case: some furniture can be driven over, but will collide with
650 // wheel parts; if the vehicle starts over such a furniture we'll mark that tile
651 // as safe and may collide with it by turning; however if we mark it unsafe
652 // we'll have no viable paths away from the starting point.
653 return &ovp->vehicle() == &driven_veh;
654 }
655
656 const tripoint_abs_ms pt_abs( here.getabs( pt ) );
657 const tripoint_abs_omt pt_omt = project_to<coords::omt>( pt_abs );
658 // only check visibility for the current OMT, we'll check other OMTs when
659 // we reach them
660 if( pt_omt == data.current_omt ) {
661 // driver must see the tile or have seen it before in order to plan a route over it
662 if( !driver.sees( pt ) ) {
663 if( !driver.is_avatar() ) {
664 return false;
665 } else if( !driver.as_avatar()->has_memorized_tile_for_autodrive( pt_abs.raw() ) ) {
666 // apparently open air doesn't get memorized, so pretend it is or else
667 // we can't fly helicopters due to the many unseen tiles behind the driver
668 if( !( data.air_ok && here.ter( pt ) == t_open_air ) ) {
669 return false;
670 }
671 }
672 }
673 }
674
675 // check for creatures
676 // TODO: padding around monsters
677 Creature *critter = g->critter_at( pt, true );
678 if( critter && driver.sees( *critter ) ) {
679 return false;
680 }
681
682 // don't drive over visible traps
683 if( here.can_see_trap_at( pt, driver ) ) {
684 return false;
685 }
686
687 // check for furniture that hinders movement; furniture with 0 move cost
688 // can be driven on
689 const furn_id furniture = here.furn( pt );
690 if( furniture != f_null && furniture.obj().movecost != 0 ) {
691 return false;
692 }
693
694 const ter_id terrain = here.ter( pt );
695 if( terrain == t_null ) {
696 return false;
697 }
698 // open air is an obstacle to non-flying vehicles; it is drivable
699 // for flying vehicles
700 if( terrain == t_open_air ) {
701 return data.air_ok;
702 }
703 const ter_t &terrain_type = terrain.obj();
704 // watercraft can drive on water
705 if( data.water_ok && terrain_type.has_flag( TFLAG_SWIMMABLE ) ) {
706 return true;
707 }
708 // remaining checks are for land-based navigation
709 if( !data.land_ok ) {
710 return false;
711 }
712 // NOLINTNEXTLINE(bugprone-branch-clone)
713 if( terrain_type.movecost <= 0 ) {
714 // walls and other impassable terrain
715 return false;
716 } else if( terrain_type.movecost == 2 || terrain_type.has_flag( "NOCOLLIDE" ) ) {
717 // terrain with neutral move cost or tagged with NOCOLLIDE will never cause
718 // collisions
719 return true;
720 } else if( terrain_type.bash.str_max >= 0 && !terrain_type.bash.bash_below ) {
721 // bashable terrain (but not bashable floors) will cause collisions
722 return false;
723 } else if( terrain_type.has_flag( TFLAG_LIQUID ) ) {
724 // water and lava
725 return false;
726 }
727 return true;
728}

765void vehicle::autodrive_controller::compute_goal_zone()
766{
767 data.goal_zone.clear();
768 coord_transformation goal_transform;
769 if( data.next_next_omt != data.next_omt ) {
770 // set the goal at the edge of next_omt and next_next_omt (in next_omt
771 // space, pointing towards next_next_omt)
772 const point next_omt_middle( OMT_SIZE + OMT_SIZE / 2, OMT_SIZE / 2 );
773 const tripoint_rel_omt omt_diff = data.next_next_omt - data.next_omt;
774 const quad_rotation rotation = to_quad_rotation( omt_diff.raw().xy() ) - data.nav_to_map.rotation;
775 goal_transform = {next_omt_middle, rotation, next_omt_middle};
776 } else {
777 // set the goal at the edge of cur_omt and next_omt (in next_omt space,
778 // pointing away from cur_omt)
779 goal_transform = { point( OMT_SIZE - 1, 0 ), quad_rotation::d0, point_zero };
780 }
781 constexpr int max_turns = NUM_ORIENTATIONS / 8 + 1;
782 const int x = 2 * OMT_SIZE - 1;
783 for( int turns = -max_turns; turns <= max_turns; turns++ ) {
784 const orientation dir = orientation::d0 + turns;
785 static_assert( NAV_MAP_SIZE_Y == OMT_SIZE, "Unexpected nav map size" );
786 for( int y = 0; y < OMT_SIZE; y++ ) {
787 const point pt( x, y );
788 const node_address addr = goal_transform.transform( pt, dir );
789 if( data.valid_position( addr ) ) {
790 data.goal_zone.insert( addr );
791 }
792 }
793 }
794 data.goal_points[0] = point( OMT_SIZE, OMT_SIZE / 2 - 1 );
795 data.goal_points[1] = goal_transform.transform( point( x, OMT_SIZE / 2 - 1 ) );
796}

848scored_address vehicle::autodrive_controller::compute_node_score( const node_address &addr,
849 const navigation_node &node ) const
850{
851 // TODO: tweak this
852 constexpr int cost_mult = 1;
853 constexpr int forward_dist_mult = 10;
854 constexpr int side_dist_mult = 8;
855 constexpr int angle_mult = 2;
856 constexpr int nearness_penalty = 15;
857 scored_address ret{ addr, 0 };
858 ret.score += cost_mult * node.cost;
859 if( node.is_goal ) {
860 return ret;
861 }
862 static const point neighbor_deltas[4] = { point_east, point_south, point_west, point_north };
863 for( const point &neighbor_delta : neighbor_deltas ) {
864 const point p = addr.get_point() + neighbor_delta;
865 if( !data.nav_bounds.contains( p ) || !data.valid_position( addr.facing_dir, p ) ) {
866 ret.score += nearness_penalty;
867 }
868 }
869 const int omt_index = std::min( std::max( addr.x / OMT_SIZE, 0 ),
870 static_cast<int>( data.goal_points.size() ) - 1 );
871 const point waypoint_delta = data.goal_points[omt_index] - addr.get_point();
872 const point goal_delta = data.goal_points.back() - addr.get_point();
873 const orientation waypoint_dir = approx_orientation( waypoint_delta.x, waypoint_delta.y );
874 ret.score += forward_dist_mult * std::abs( goal_delta.x );
875 ret.score += side_dist_mult * std::abs( goal_delta.y );
876 if( waypoint_delta.x > OMT_SIZE / 4 ) {
877 ret.score += angle_mult * std::abs( orientation_diff( addr.facing_dir, waypoint_dir ) );
878 }
879 return ret;
880}

883void vehicle::autodrive_controller::compute_next_nodes( const node_address &addr,
884 const navigation_node &node, int target_speed_tps,
885 std::vector<std::pair<node_address, navigation_node>> &next_nodes )
886const
887{
888 constexpr int move_cost = 10;
889 constexpr int steering_cost = 1;
890 const int sign = target_speed_tps > 0 ? 1 : -1;
891 const int target_speed = target_speed_tps * VMIPH_PER_TPS;
892 const int cur_omt = addr.x / OMT_SIZE;
893 int next_speed = target_speed;
894 int num_tiles_to_move = std::abs( target_speed_tps );
895 if( target_speed_tps > 1 && node.speed < target_speed ) {
896 const int cur_tps = std::min( std::max( node.speed / VMIPH_PER_TPS, 0 ), data.max_speed_tps - 1 );
897 next_speed = std::min( std::max<int>( node.speed, 0 ) + data.acceleration[cur_tps], target_speed );
898 num_tiles_to_move = next_speed / VMIPH_PER_TPS;
899 }
900 for( int steer = -data.max_steer; steer <= data.max_steer; steer++ ) {
901 node_address next_addr = addr;
902 next_addr.facing_dir = addr.facing_dir + steer;
903 tileray tdir = data.profile( next_addr.facing_dir ).tdir;
904 if( steer == 0 ) {
905 tdir.advance( node.tileray_steps );
906 }
907 bool ok = true;
908 bool goal_found = false;
909 for( int i = 0; i < num_tiles_to_move && ok; i++ ) {
910 tdir.advance( sign );
911 next_addr.x += tdir.dx();
912 next_addr.y += tdir.dy();
913 if( !data.nav_bounds.contains( next_addr.get_point() ) ) {
914 // it's ok to go out of bounds only from the "next" omt and only
915 // after crossing the goal zone
916 if( cur_omt == 0 || !goal_found ) {
917 ok = false;
918 }
919 } else if( !data.valid_position( next_addr ) ) {
920 ok = false;
921 } else if( !goal_found && data.goal_zone.find( next_addr ) != data.goal_zone.end() ) {
922 goal_found = true;
923 }
924 }
925 if( !ok ) {
926 continue;
927 }
928 const int next_omt = next_addr.x / OMT_SIZE;
929 if( next_omt < cur_omt ) {
930 // no going back to a previous omt
931 continue;
932 }
933 navigation_node next_node;
934 next_node.prev = addr;
935 next_node.tileray_steps = num_tiles_to_move + ( !steer ? node.tileray_steps : 0 );
936 // TODO: improve this
937 next_node.cost = node.cost + move_cost * num_tiles_to_move + ( steer ? steering_cost : 0 );
938 next_node.speed = next_speed;
939 next_node.target_speed_tps = target_speed_tps;
940 next_node.is_goal = goal_found;
941 next_nodes.emplace_back( next_addr, next_node );
942 }
943}
944
945cata::optional<std::vector<navigation_step>> vehicle::autodrive_controller::compute_path(
946 int speed_tps ) const
947{
948 if( speed_tps == 0 || speed_tps < -1 ) {
949 return cata::nullopt;
950 }
951 // TODO: tweak this
952 constexpr size_t max_search_count = 10000;
953 std::vector<navigation_step> ret;
954 // TODO: check simple reachability first and bail out or set upper bound on node score
955 std::unordered_map<node_address, navigation_node, node_address_hasher> known_nodes;
956 std::priority_queue<scored_address, std::vector<scored_address>, std::greater<>>
957 open_set;
958 const tripoint_abs_ms veh_pos = driven_veh.global_square_location();
959 const node_address start = data.nav_to_map.inverse().transform(
960 veh_pos.raw().xy(), to_orientation( driven_veh.face.dir() ) );
961 known_nodes.emplace( start, make_start_node( start, driven_veh ) );
962 open_set.push( scored_address{ start, 0 } );
963 std::vector<std::pair<node_address, navigation_node>> next_nodes;
964 while( !open_set.empty() ) {
965 const node_address cur_addr = open_set.top().addr;
966 open_set.pop();
967 const navigation_node &cur_node = known_nodes[cur_addr];
968 if( cur_node.is_goal ) {
969 node_address addr = cur_addr;
970 while( !( addr == start ) ) {
971 const navigation_node &node = known_nodes.at( addr );
972 const node_address &prev = node.prev;
973 const tripoint_abs_ms prev_loc( data.nav_to_map.transform( prev.get_point(),
974 data.current_omt.z() ) );
975 ret.emplace_back( navigation_step{
976 prev_loc,
977 data.nav_to_map.transform( addr.facing_dir ),
978 node.target_speed_tps
979 } );
980 addr = prev;
981 }
982 return ret;
983 }
984 next_nodes.clear();
985 compute_next_nodes( cur_addr, cur_node, speed_tps, next_nodes );
986 for( const auto &next : next_nodes ) {
987 const node_address &next_addr = next.first;
988 const navigation_node &next_node = next.second;
989 auto iter = known_nodes.find( next_addr );
990 if( iter != known_nodes.end() ) {
991 const navigation_node &other_node = iter->second;
992 if( next_node.cost < other_node.cost ) {
993 const bool same_dir = cur_addr.facing_dir == next_addr.facing_dir;
994 const bool dir_multiple_45 = static_cast<int>( next_addr.facing_dir ) % 3 == 0;
995 if( ( !same_dir && next_node.tileray_steps == 1 ) || dir_multiple_45 ) {
996 iter->second = next_node;
997 }
998 }
999 } else if( known_nodes.size() < max_search_count ) {
1000 known_nodes[next_addr] = next_node;
1001 open_set.push( compute_node_score( next_addr, next_node ) );
1002 }
1003 }
1004 }
1005 return cata::nullopt;
1006}

1025
1026collision_check_result vehicle::autodrive_controller::check_collision_zone( orientation turn_dir )
1027{
1028 const tripoint veh_pos = driven_veh.global_pos3();
1029
1030 // first check if we have any visibility in front, to prevent blind driving
1031 tileray face_dir = driven_veh.face;
1032 face_dir.advance();
1033 const point forward_offset( face_dir.dx(), face_dir.dy() );
1034 bool blind = true;
1035 for( const point &p : data.profile( to_orientation( face_dir.dir() ) ).collision_points ) {
1036 if( driver.sees( veh_pos + forward_offset + p ) ) {
1037 blind = false;
1038 }
1039 }
1040 if( blind ) {
1041 return collision_check_result::no_visibility;
1042 }
1043
1044 // now check the area we're about to move into in 1 step, anything there
1045 // should force us to stop
1046 const vehicle_profile &profile = data.profile( turn_dir );
1047 tileray tdir = driven_veh.face;
1048 if( turn_dir != to_orientation( tdir.dir() ) ) {
1049 tdir = profile.tdir;
1050 }
1051 const int speed = std::min( driven_veh.velocity + driven_veh.current_acceleration(),
1052 driven_veh.cruise_velocity );
1053 const int speed_tps = speed / VMIPH_PER_TPS;
1054 std::unordered_set<point> collision_zone;
1055 tdir.advance();
1056 point offset( tdir.dx(), tdir.dy() );
1057 for( const point &p : profile.occupied_zone ) {
1058 collision_zone.insert( p + offset );
1059 }
1060 for( const point &p : collision_zone ) {
1061 if( !check_drivable( veh_pos + p ) ) {
1062 return collision_check_result::close_obstacle;
1063 }
1064 }
1065
1066 // finally check the area further ahead; we can still avoid those collisions by reducing speed
1067 collision_zone.clear();
1068 for( int i = 1; i < speed_tps; i++ ) {
1069 tdir.advance();
1070 offset += point( tdir.dx(), tdir.dy() );
1071 for( const point &p : profile.collision_points ) {
1072 collision_zone.insert( p + offset );
1073 }
1074 }
1075 for( const point &p : collision_zone ) {
1076 if( !driver.sees( veh_pos + p ) ) {
1077 return collision_check_result::slow_down;
1078 }
1079 if( !check_drivable( veh_pos + p ) ) {
1080 return collision_check_result::slow_down;
1081 }
1082 }
1083 return collision_check_result::ok;
1084}

1090
1091cata::optional<navigation_step> vehicle::autodrive_controller::compute_next_step()
1092{
1093 precompute_data();
1094 const tripoint_abs_ms veh_pos = driven_veh.global_square_location();
1095 const bool had_cached_path = !data.path.empty();
1096 while( !data.path.empty() && data.path.back().pos != veh_pos ) {
1097 data.path.pop_back();
1098 }
1099 if( !data.path.empty() && data.path.back().target_speed_tps > data.max_speed_tps ) {
1100 data.path.clear();
1101 }
1102 if( data.path.empty() ) {
1103 // if we're just starting out or we've gone off-course use the lowest speed
1104 if( had_cached_path || driven_veh.velocity == 0 ) {
1105 data.max_speed_tps = MIN_SPEED_TPS;
1106 }
1107 auto new_path = compute_path( data.max_speed_tps );
1108 while( !new_path && data.max_speed_tps > MIN_SPEED_TPS ) {
1109 // high speed didn't work, try a lower speed
1110 data.max_speed_tps /= 2;
1111 new_path = compute_path( data.max_speed_tps );
1112 }
1113 if( !new_path ) {
1114 return cata::nullopt;
1115 }
1116 data.path.swap( *new_path );
1117 }
1118 return data.path.back();

1214 }
1215 const tripoint_abs_ms veh_pos = global_square_location();
1216 const tripoint_abs_omt veh_omt = project_to<coords::omt>( veh_pos );
1217 std::vector<tripoint_abs_omt> &omt_path = driver.omt_path;

1219 omt_path.pop_back();
1220 }
1221 if( omt_path.empty() ) {
1222 stop_autodriving( false );
1223 return autodrive_result::finished;
1224 }

1227 }

1229 debugmsg( "Driver changed while auto-driving" );
1230 stop_autodriving();

1232 }
1233 active_autodrive_controller->check_safe_speed();
1234 cata::optional<navigation_step> next_step = active_autodrive_controller->compute_next_step();
1235 if( !next_step ) {
1236 // message handles pathfinding failure either due to obstacles or inability to see
1237 driver.add_msg_if_player( _( "Can't see a path forward." ) );
1238 stop_autodriving( false );
1239 return autodrive_result::abort;
1240 }
1241 if( next_step->pos != veh_pos ) {
1242 debugmsg( "compute_next_step returned an invalid result" );
1243 stop_autodriving();
1244 return autodrive_result::abort;
1245 }
1246 if( next_step->target_speed_tps == 0 && velocity == 0 ) {
1247 stop_autodriving( false );
1248 return autodrive_result::finished;
1249 }
1250 cruise_on = true;
1251 cruise_velocity = next_step->target_speed_tps * VMIPH_PER_TPS;
1252
1253 // check for collisions before we steer, since steering may end our turn
1254 // which would cause the vehicle to move and maybe crash
1255 switch( active_autodrive_controller->check_collision_zone( next_step->steering_dir ) ) {
1256 case collision_check_result::no_visibility:
1257 driver.add_msg_if_player( m_warning, _( "You can't see anything in front of your vehicle!" ) );

1261 driver.add_msg_if_player( m_warning, _( "You're about to crash into something!" ) );
1262 stop_autodriving( false );
1263 return autodrive_result::abort;
1264 case collision_check_result::slow_down:
1265 active_autodrive_controller->reduce_speed();
1266 if( cruise_velocity > VMIPH_PER_TPS ) {
1267 cruise_velocity = VMIPH_PER_TPS;
1268 }
1269 break;
1270 case collision_check_result::ok:
1271 break;
1272 }
1273
1274 int turn_delta = orientation_diff( next_step->steering_dir, to_orientation( turn_dir ) );
1275 // pldrive() does not handle steering multiple times in one call correctly
1276 // call it multiple times, matching how a player controls the vehicle
1277 for( int i = 0; i < std::abs( turn_delta ); i++ ) {
1278 if( driver.moves <= 0 ) {
1279 // we couldn't steer as many times as we wanted to but there's
1280 // nothing we can do about it now, hope we don't crash!
1281 break;
1282 }
1283 pldrive( driver, { signum( turn_delta ), 0 } );
1284 }
1285 // Don't do anything else below; the driver's turn may be over (moves <= 0) so
1286 // any extra actions would be "cheating".
1287
1288 return autodrive_result::ok;
1289}
