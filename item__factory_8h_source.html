94 * uses the ident and subtype given as parameter and does not look up those things in

98 * @param jsobj The json object to load from.
99 * @param group_id The ident of the item that is to be loaded.
100 * @param subtype The type of the item group, either "collection", "distribution" or "old"
101 * ("old" is a distribution, too).
102 * @throw JsonError if the json object contains invalid data.
103 */

105 const std::string &subtype );
106 /**
107 * Like above, but the above loads data from several members of the object, this function
108 * assume the given array is the "entries" member of the item group.
109 *
110 * For each element in the array, @ref Item_factory::add_entry is called.
111 *
112 * Assuming the input array looks like `[ x, y, z ]`, this function loads it like the
113 * above would load this object:
114 * \code
115 * {
116 * "subtype": "depends on is_collection parameter",
117 * "id": "identifier",
118 * "entries": [ x, y, z ]
119 * }
120 * \endcode
121 * Note that each entry in the array has to be a JSON object. The other function above
122 * can also load data from arrays of strings, where the strings are item or group ids.
