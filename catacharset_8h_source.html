61/**
62 * UTF8-Wrapper over std::string.
63 * It looks and feels like a std::string, but uses code points counts
64 * as index, not bytes.
65 * A multi-byte Unicode character might be represented
66 * as 3 bytes in UTF8, this class will see these 3 bytes as 1 character.
67 * It will never separate them. It will however split between code points
68 * which might be problematic when containing combination characters.
69 * In this case use the *_display functions. They operate on the display width.
70 * Code points with a zero width are considered to belong to the previous code
71 * point and are not split from that.
72 * Having a string with like [letter0][letter1][combination-mark][letter2]
73 * (assuming each letter has a width of 1) will return a display_width of 3.
74 * substr_display(1, 2) returns [letter1][combination-mark][letter2],
75 * substr_display(1, 1) returns [letter1][combination-mark]
76 * substr_display(2, 1) returns [letter2]
77 *
78 * Note: functions use code points, not bytes, for counting/indexing!
79 * Functions with the _display suffix use display width for counting/indexing!
80 * Protected functions might behave different
81 *
82 * For function documentation see std::string, the functions here
83 * mimic the behavior of the equally named std::string function.
84 */
